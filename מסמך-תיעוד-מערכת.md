# מסמך תיעוד מערכת – פורטל ידע מבצעי חמ"ם 7134

**תאריך:** מרץ 2025  
**פרויקט:** פורטל צוות אלפא 7134 (פרוייקט מתנ"ה)

---

## 1. סקירה כללית

אפליקציית ווב React (Vite) בעברית (RTL), פורטל פנימי ליחידה. כולל דף בית עם ניווט דינמי, מופעי חודש, דבר המפקד, וממשק ניהול (אירועים + ניווט). נתונים נשמרים ב-**localStorage** במצב פיתוח (Mock) או ב-**SharePoint** (קבצי JSON ב-SiteAssets) בייצור.

## 3. מקור האפליקציה (`src/`)

### 3.1 כניסה ו-CSS

| קובץ | תיאור |
|------|--------|
| `src/main.jsx` | נקודת כניסה: `createRoot`, `BrowserRouter`, `AuthProvider` → `NavigationProvider` → `EventsProvider` → `App`. |
| `src/index.css` | גופן Heebo (Google Fonts), Tailwind base/components/utilities, `direction: rtl`, רקע `#0c0d12`, גריד רקע (`.grid-overlay`), scrollbar מותאם, `.custom-scrollbar`, `.mask-image-bottom`, אנימציה `aggressiveEntrance` / `.animate-aggressive`. |
| `src/App.css` | סטיילים ישנים (max-width 1280px, .logo, .card וכו') – חלקם כנראה לא בשימוש פעיל. |

### 3.2 `src/App.jsx` – דף בית ו-Routing

- **Routes:**  
  - `"/"` → `Home`  
  - `"/admin/*"` → `AdminHub`
- **Home:**  
  - רקע מתחלף כל 3 שניות מ-6 תמונות מ-`/images/לח*.{jpeg,jpg,webp}`.  
  - Navbar: לוגו "בית הספר", קישורי ניווט (מ-NavigationContext), שדה חיפוש (ללא לוגיקה), כפתור "ניהול", ברכת שלום + שם משתמש (מ-AuthContext).  
  - Hero: לוגו + "צוות אלפא 7134", טקסט ברוכים הבאים, תיאור יחידה.  
  - שני פאנלים בתחתית:  
    - **דבר המפקד:** תמונת פורטרט (`/images/פורטרט.png`), כותרת, ציטוט קבוע, "סא"ל א', מפקד בית הספר", כפתורי ניווט (עיצוב בלבד, ללא לוגיקה).  
    - **מופעי החודש:** רכיב `EventsList` (גלילה, רוטציה אוטומטית).  
  - מתחת ל-Hero: סקשנים דינמיים לפי `navItems` – רק קטגוריות עם `children` ולייט `url`. כל סקשן: כותרת עם אייקון, גריד של **FlipCard**.  
- **FlipCard (export מתוך App.jsx):**  
  - כרטיס עם צד קדמי (אייקון + כותרת + "לכניסה") וצד אחורי (כותרת + רשימת `subLinks`).  
  - קליק: אם יש `url` – פתיחה בחלון חדש; אחרת היפוך כרטיס.  
  - `subLinks`: כפתורים עם אייקון + label; אם יש `link.url` – לינק מוחלט.  
- **ניווט אנכי:** `handleNavTo(cat)` – אם `cat.url` פתיחה בחלון חדש, אחרת scroll ל-`#cat.id`.

### 3.3 קומפוננטות (`src/components/`)

| קומפוננטה | תיאור |

### 3.4 Contexts (`src/context/`)

| Context | תיאור |
|---------|--------|
| **AuthContext.jsx** | `currentUser`, `isAdmin`, `loading`, `signIn`, `signOut`, `trySharePointLogin`. טעינת רשימת משתמשים מ-UsersService; בדיקת `sessionStorage['tracker_user_name']`; אם אין – ניסיון SharePoint `/_api/web/currentuser`. אדמין: `SHAREPOINT_CONFIG.useMock` או שם ברשימת המשתמשים. |

### 3.5 שירותים (`src/services/`)

| שירות | תיאור |
|--------|--------|
| **EventsService.js** | `getEvents()` → `{ displayCount, events }`, `saveEvents(payload)`. Mock: localStorage (`bihs_events_data`). Production: SharePoint – קריאה/כתיבה לקובץ ב-`fileServerRelativeUrl`. מיגרציה לאירועים ישנים (day/month → date). אימות אחרי שמירה. |
| **NavigationService.js** | `getNavigation()`, `saveNavigation(navData)`. Mock: localStorage (`bihs_nav_data`) + seed עם קטגוריות (הכשרות, אג"ם, מפקדה, גרף יחידה, תיקי יסוד, גלריית יחידה, בטיחות). Production: SharePoint – קובץ ב-`navFileServerRelativeUrl`. |
| **UsersService.js** | `getUsers()`, `saveUsers(usersData)`. Mock: localStorage (`bihs_users_data`) + ברירת מחדל (מני, Admin, מנהל). Production: SharePoint – קובץ ב-`usersFileServerRelativeUrl`; ב-404 מחזיר רשימת ברירת מחדל ומנסה לשמור. |

## 4. זרימת נתונים

1. **אתחול:**  
   AuthProvider טוען משתמשים (UsersService) ובודק session / SharePoint.  
   NavigationProvider ו-EventsProvider טוענים ניווט ואירועים מהשירותים.

2. **דף הבית:**  
   קורא `navItems` מ-NavigationContext ו-`currentUser` מ-AuthContext. מציג EventsList מ-EventsContext ו-FlipCards לפי קטגוריות וילדים.

3. **ממשק ניהול:**  
   AdminEvents ו-AdminNavigation מעדכנים state מקומי ואז קוראים ל-`saveEvents` / `saveNavigation` ב-Context. השירותים כותבים ל-localStorage (Mock) או ל-SharePoint.

4. **גיבוי:**  
   רק ב-Production; מ-AdminHub קוראים ל-`createBackup()` שמעתיק את קבצי הנתונים לתיקיית Backups.

## 6. אייקונים (Lucide)

המערכת משתמשת ב-**lucide-react**. שמות אייקונים מופיעים כמחרוזות בנתוני ניווט (למשל Rocket, GraduationCap, Users, Calendar, FileText, Target, BarChart, Briefcase, Camera, AlertTriangle, Map, Crosshair וכו') ומוצגים דרך `DynamicIcon`.

## 7. אבטחה והרשאות

- **התחברות:** שם משתמש ב-sessionStorage; במצב Production אפשרות זיהוי אוטומטי מ-SharePoint.  
- **אדמין:** רשימת שמות מ-UsersService (או Mock); במצב Mock כל המשתמשים נחשבים אדמין.  
- **ממשק ניהול:** כפתור "ניהול" מוביל ל-`/admin` – אין בדיקת `isAdmin` לפני כניסה ל-AdminHub (מומלץ להוסיף הגנה לפי תפקיד אם נדרש).
